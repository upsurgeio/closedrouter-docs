## Overview

Pineapple provides a unified API that gives you access to multiple AI features through a single endpoint, while automatically handling feature selection and model routing. Get started with just a few lines of code.

**Note**: All features—including structured outputs, tool calling, and multimodality—are determined via the request body, not separate endpoints. The common endpoint for all requests is:

```
POST /chat/completions
```

## Using Pineapple API Directly

<CodeGroup>

```python
import requests
import json

response = requests.post(
  url="https://api.closedrouter.ai/v1/chat/completions",
  headers={
    "Authorization": "Bearer <PINEAPPLE_API_KEY>",
    "Content-Type": "application/json"
  },
  json={
    "model": "openai/gpt-4o",
    "messages": [
      {
        "role": "user",
        "content": "Extract the user's name, email, and subscription type from this message: 'Hi, I am Alice, my email is alice@example.com and I have a premium subscription.'"
      }
    ],
    "response_format": {
      "type": "json_schema",
      "json_schema": {
        "name": "user_info",
        "strict": true,
        "schema": {
          "type": "object",
          "properties": {
            "name": {"type": "string", "description": "User's full name"},
            "email": {"type": "string", "description": "User's email address"},
            "subscription": {"type": "string", "description": "Subscription type"}
          },
          "required": ["name", "email", "subscription"],
          "additionalProperties": false
        }
      }
    }
  }
)

result = response.json()
print(result["choices"][0]["message"]["content"])
```

```javascript
const response = await fetch('https://api.closedrouter.ai/v1/chat/completions', {
  method: 'POST',
  headers: {
    Authorization: 'Bearer <PINEAPPLE_API_KEY>',
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    model: 'openai/gpt-4o',
    messages: [
      {
        role: 'user',
        content: 'Extract the user info from this message: "Hi, I am Alice, my email is alice@example.com and I have a premium subscription."'
      }
    ],
    response_format: {
      type: 'json_schema',
      json_schema: {
        name: 'user_info',
        strict: true,
        schema: {
          type: 'object',
          properties: {
            name: { type: 'string', description: "User's full name" },
            email: { type: 'string', description: "User's email address" },
            subscription: { type: 'string', description: "Subscription type" }
          },
          required: ['name', 'email', 'subscription'],
          additionalProperties: false
        }
      }
    }
  })
});

const result = await response.json();
console.log(result.choices[0].message.content);
```

## Key Features

### 1. **Single Endpoint**
- Use `/chat/completions` for all AI features
- No need to learn multiple endpoints or APIs

### 2. **Structured Outputs**
- Use `response_format.json_schema` to enforce JSON schema validation
- Get consistent, machine-readable responses every time
- Perfect for production pipelines and integrations

### 3. **Tool Calling**
- Enable function calling with `tools` parameter
- Specify which tools the model can use
- Automatic tool selection based on context

### 4. **Multimodality**
- Support for text, images, and other media types
- Combined input/output processing
- Seamless handling of multiple content types

### 5. **Model Routing**
- Automatic model selection for optimal performance
- Override with specific model names when needed
- Support for OpenAI, Anthropic, and other providers
